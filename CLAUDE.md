## Architecture decisions (settled)

- **ADR-001:** Frontend is Astro вЂ” chosen for minimal client-side JS, best Core Web Vitals and SEO for a content catalog.
- **ADR-002:** Docs-as-code вЂ” documentation lives in `/docs` as Markdown committed alongside code. Notion is for planning/goals only, not technical docs.
- **ADR-003:** ETL is Apache NiFi вЂ” self-hosted on existing Oracle Cloud infra. NiFi flows are exported as JSON and version-controlled in `/nifi-flows/`.
- **ADR-004:** Hosting is Cyberfolks + Cloudflare вЂ” Cloudflare must be configured before launch (not after).

## Open architectural decisions

- **ADR-005:** Backend language вЂ” Node.js vs Python (FastAPI). Evaluate based on AI/scraping library ecosystem.
- **ADR-006:** Admin panel вЂ” Supabase (fast start, built-in auth) vs Directus (headless CMS, more control) vs custom.

## Data model

Core `companies` table:

```sql
CREATE TABLE companies (
  id          SERIAL PRIMARY KEY,
  name        VARCHAR(255) NOT NULL,
  slug        VARCHAR(255) UNIQUE NOT NULL,
  description TEXT,                          -- AI-generated
  url         VARCHAR(500),
  category_id INT REFERENCES categories(id),
  tags        TEXT[],
  pricing     VARCHAR(50),                   -- free/freemium/paid/enterprise
  tier        VARCHAR(20) DEFAULT 'free',    -- free/premium (monetization)
  status      VARCHAR(20) DEFAULT 'pending', -- pending/approved/rejected
  added_at    TIMESTAMP DEFAULT NOW(),
  updated_at  TIMESTAMP DEFAULT NOW()
);
```

## SEO conventions

- URLs: `/narzedzia/[slug]` and `/kategoria/[slug]`
- Every page requires: `title`, `meta description`, OG tags, `schema.org` (SoftwareApplication)
- Sitemap: auto-generated by Astro
- SSR for dynamic pages, SSG for static pages

## Code conventions

- File names: `kebab-case` (e.g., `company-card.astro`)
- Components: `PascalCase` (e.g., `CompanyCard`)
- Variables: `camelCase`
- SQL keywords: `UPPER_CASE`
- Git commit prefixes: `feat:`, `fix:`, `docs:`, `refactor:`
